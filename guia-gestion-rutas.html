Gestionar la web sin salir del navegador

Todo lo que se puede cambiar en la web — rutas, textos, imágenes, colores — se hace desde github.com, la página donde vive el proyecto. No hace falta instalar nada, no hace falta saber programar, y no hace falta la aplicación de escritorio de GitHub. Esta guía explica cada tarea paso a paso, botón a botón.

Antes de empezar

Cinco ideas que hacen falta para entender todo lo demás. Léelas una vez y ya no hace falta volver.

RepositorioLa carpeta del proyecto entero, alojada en GitHub. La de esta web es vgonzcam/ownGpxRutasApp — todo lo que hay dentro (rutas, textos, colores) vive ahí.
ArchivoUn documento suelto dentro del repositorio: un .gpx, una imagen, o un archivo de texto como config.json.
CommitGuardar un cambio, con un mensaje corto explicando qué hiciste. Cada vez que subes, editas o borras algo en GitHub, al final te pide "hacer commit" — es el equivalente a pulsar Guardar.
Rama mainLa versión oficial del sitio, la que está publicada ahora mismo. Todos los pasos de esta guía trabajan directamente sobre main — no hace falta crear ramas ni entender qué son.
GitHub ActionsEl robot que, después de cada commit, coge el contenido de main y lo publica en vgonzcam.github.io/ownGpxRutasApp. Tarda 1-2 minutos. Ver el Patrón D.
Acceso necesario: hace falta una cuenta de GitHub añadida como colaboradora de este repositorio con permiso de escritura ("Write"). Si al intentar guardar un cambio no ves el botón verde de commit sino algo sobre "crear una rama" o "fork", es que tu cuenta todavía no tiene ese permiso — pídeselo a quien administra el repositorio, en Settings → Collaborators.

Los cuatro movimientos que se repiten siempre

Todas las tareas de esta guía son combinaciones de estos cuatro patrones. Apréndelos aquí una vez; las recetas de más abajo solo dicen "sigue el Patrón A" y qué cambiar exactamente.

AEditar un texto

Cambiar una línea dentro de config.json o theme.json

Sirve para cualquier texto, número o color: el título del sitio, un email, una etiqueta de dificultad, un color del tema...

  1. 1Entra en github.com/vgonzcam/ownGpxRutasApp y navega hasta el archivo (por ejemplo rutas/config.json).
  2. 2Arriba a la derecha del contenido del archivo, pulsa el icono del lápiz ✏️ ("Edit this file").
  3. 3Busca la línea que quieres cambiar. Es más rápido con Ctrl F / Cmd F del navegador que leyendo todo el archivo.
  4. 4Escribe el valor nuevo entre las mismas comillas que ya estaban. No toques comas, llaves { } ni corchetes [ ] de alrededor.Estos archivos son JSON: una coma de más o de menos, o una comilla sin cerrar, rompe la web entera. Ver Si algo falla.
  5. 5Baja al final de la página → sección Commit changes → escribe un mensaje corto (p. ej. "Actualiza el email de contacto").
  6. 6Deja marcada la opción ◉ Commit directly to the main branch y pulsa Commit changes.
Sigue con el Patrón D para comprobar que ya está en la web.
BSubir un archivo

Añadir un .gpx nuevo, o el logo de un patrocinador

Sirve para cualquier archivo nuevo: una ruta, una foto para el banner, el logo de un patrocinador.

  1. 1Navega hasta la carpeta donde tiene que quedar el archivo (por ejemplo rutas/GRAVEL, o imgs).
  2. 2Botón Add file ▾ → Upload files.
  3. 3Arrastra el archivo desde tu ordenador a la zona punteada, o pulsa "choose your files".
  4. 4Baja al final → mensaje corto → ◉ Commit directly to the main branch → Commit changes.
Sigue con el Patrón D para comprobar que ya está en la web.
CEliminar un archivo

Quitar una ruta o una imagen para siempre

  1. 1Abre el archivo que quieres borrar (clica en su nombre dentro de la carpeta).
  2. 2Arriba a la derecha, icono de la papelera 🗑 ("Delete this file").
  3. 3Baja al final → mensaje corto (p. ej. "Elimina ruta antigua") → ◉ Commit directly to the main branch → botón rojo Commit changes.
No hace falta tocar nada más — si esa ruta tenía una entrada en routeOverrides (ver Corregir los datos), puedes dejarla o borrarla, la web la ignora si el archivo ya no existe.
DPublicar y deshacer

Comprobar que un cambio ya está en la web — y volver atrás si algo se rompe

Publicar: cada commit dispara el robot de despliegue automáticamente. No hay ningún botón de "publicar" aparte.

  1. 1En el repositorio, pestaña Actions (arriba, junto a "Code").
  2. 2La entrada más reciente muestra un círculo amarillo (en curso) y luego un check verde ✅ (terminado) o una cruz roja ❌ (falló).Suele tardar entre 1 y 2 minutos.
  3. 3Abre la web y recarga forzado con Ctrl Shift R (Windows) o Cmd Shift R (Mac) — la web guarda caché para funcionar sin conexión, y a veces hay que forzar la recarga para ver lo último.

Deshacer: todo archivo en GitHub guarda su historial completo. Si un cambio rompe algo, puedes volver a la versión de antes sin arreglar nada a mano.

  1. 1Abre el archivo afectado (normalmente rutas/config.json) y pulsa History (arriba a la derecha del contenido, icono de reloj).
  2. 2Verás la lista de commits anteriores. Clica en el que quieras recuperar — el de justo antes del que rompió algo.
  3. 3Arriba a la derecha de esa versión antigua, botón ... ▾ → Restore this file (o copia su contenido y pégalo editando el archivo actual, Patrón A).

Gestionar rutas

Los tipos de ruta (MTB, ROAD, GRAVEL...) no son una lista fija en el código: son, literalmente, las carpetas que hay dentro de rutas/.

01 · Añadir una ruta

Cuando el club graba o recibe un .gpx nuevo de un tipo que ya existe en la web (MTB, ROAD, GRAVEL, TRAIL, BIKEPACKING, CASTELLBIKE).

BSubir un archivo
  1. 1Ve a rutas/ → entra en la carpeta del tipo (p. ej. rutas/ROAD).
  2. 2Sigue el Patrón B con tu archivo .gpx.El nombre y la ubicación de la ruta se leen del propio GPX (etiqueta <name>Nombre, Ubicación</name>), así que expórtalo con un nombre descriptivo antes de subirlo.
La distancia, el desnivel, la duración y la dificultad se calculan solos a partir del track. Si algún dato no es correcto, no hace falta volver a subir el archivo: ver Corregir los datos.

02 · Añadir un tipo de ruta nuevo

Cuando el club empieza una modalidad que no encaja en ninguna carpeta actual (por ejemplo "XC" o "URBAN").

BSubir un archivo
  1. 1Entra en rutas/ (no en ninguna subcarpeta) → Add file ▾ → Upload files.
  2. 2Arrastra el primer .gpx de ese tipo. Encima de la zona de subida aparece un campo de texto con la ruta de destino (algo como rutas/) — edítalo para añadir el nombre de la carpeta nueva delante del archivo, en mayúsculas para que coincida con el resto: rutas/XC/mi-primera-ruta-xc.gpx.
  3. 3Mensaje corto → ◉ Commit directly to the main branch → Commit changes. GitHub crea la carpeta sola, en el mismo paso, porque no existía.
El tipo nuevo aparece solo en el desplegable de filtros, al final de la lista. Si quieres decidir en qué posición sale, añade su nombre a filters.trackTypes en config.json — ver referencia ↗.

03 · Eliminar una ruta

Cuando una ruta ya no es válida, está duplicada, o se subió por error.

CEliminar un archivo
  1. 1Ve a rutas/ → carpeta del tipo → abre el archivo .gpx que quieres quitar.
  2. 2Sigue el Patrón C.
No hay papelera de reciclaje: si necesitas recuperar el archivo más adelante, está en el historial de la carpeta (History), pero desaparece de la web en el siguiente despliegue.

04 · Corregir los datos de una ruta

Cuando la dificultad, la distancia, el nombre o cualquier otro dato calculado automáticamente del GPX no es correcto — sin tener que volver a exportar ni resubir el archivo.

AEditar un texto

Estos datos se calculan solos a partir de los puntos del track en cuanto subes el .gpx. Para fijar cualquiera de ellos a mano, se añade una entrada dentro de routeOverrides, en rutas/config.json. El dato que pongas ahí manda siempre sobre el cálculo — solo hace falta escribir los campos que quieras corregir, el resto se sigue calculando solo.

  1. 1Averigua la clave exacta de la ruta: es carpeta/nombre-del-archivo.gpx, tal como aparece en rutas/<TIPO>. Por ejemplo ROAD/rc23-road-llarga-150k.gpx.
  2. 2Abre rutas/config.json y sigue el Patrón A hasta el bloque "routeOverrides": { ... }.
  3. 3Añade (o edita) una entrada con esa clave y los campos que quieras corregir.
−"routeOverrides": {}
+"routeOverrides": { "ROAD/rc23-road-llarga-150k.gpx": { "difficulty": "dificil", "distanceKm": 152 } }

Si ya existen otras rutas dentro de routeOverrides, añade una coma después del } de la anterior y pega tu bloque nuevo detrás — sigue siendo una sola llave routeOverrides con varias rutas dentro.

CampoCorrigeFormato del valor
nameNombre de la rutaTexto libre, entre comillas
locationUbicaciónTexto libre, entre comillas
difficultyDificultatExactamente "facil", "dificil" o "moltDificil"
durationDurada estimadaTexto libre, p. ej. "5h 30m"
distanceKmDistànciaNúmero, sin comillas ni unidad (km), p. ej. 152
elevGainDesnivell positiuNúmero en metros, p. ej. 3900
elevLossDesnivell negatiuNúmero en metros
maxAltAltitud màximaNúmero en metros
minAltAltitud mínimaNúmero en metros

06 · Banner y enlaces

La caja grande de la pantalla de inicio (imagen, título, texto, botón) y los enlaces rápidos que salen debajo ("Enllaços").

AEditar un texto BSubir un archivo

Ambos viven en sections, una lista donde cada elemento es un bloque "type": "banner" (solo puede haber uno, el primero) o "type": "link" (puede haber varios).

{ "type": "banner", "title": "Club Esportiu Castelldans", "body": "Texto largo del club...", "image": "club.avif", "linkLabel": "Vols venir?", "linkUrl": "https://cecastelldans.wixsite.com/..." }, { "type": "link", "label": "Botiga", "url": "https://cecastelldans.wixsite.com/inici/maillot" }
Para cambiar la foto del banner: sube la imagen nueva a imgs/ con el Patrón B, y luego cambia el valor de image por el nombre exacto de ese archivo (Patrón A). Si image no existe o el archivo falla al cargar, se muestra el title en texto en su lugar — nunca se rompe.

07 · Patrocinadores

Añadir, quitar o reemplazar un patrocinador, u ocultar la sección entera sin borrar la lista.

AEditar un texto BSubir un archivo
  1. 1Logo nuevo: súbelo a imgs/patrocinadors/ con el Patrón B.
  2. 2Abre rutas/config.json (Patrón A) y añade una entrada en sponsors:El campo logo es opcional — si lo omites, o el archivo falla al cargar, se muestra el name en texto.
{ "name": "Nuevo Patrocinador", "logo": "patrocinadors/nuevo.avif" }

Para quitar uno, borra su bloque entero (con su coma) dentro de sponsors. Para ocultar la sección sin perder la lista, pon "showSponsors": false en la raíz del archivo.

−"showSponsors": true,
+"showSponsors": false,

08 · Redes sociales

Añadir, quitar o cambiar el usuario mostrado de una red social en la caja "Segueix-nos" (solo en inicio).

AEditar un texto

Cada entrada de social lleva una red (instagram, x, facebook, strava o youtube) y su URL. El usuario que se muestra se detecta solo del final de la URL — si quieres forzar un texto distinto, añade label.

{ "network": "instagram", "url": "https://www.instagram.com/garriguesbike/", "label": "Garrigues Bike" }

Una red no reconocida (fuera de esas cinco) se muestra igualmente, con un icono genérico.

Apariencia

09 · Colores del sitio

Cambiar la paleta de colores de toda la web, en modo claro y modo oscuro a la vez.

AEditar un texto

Los colores no están en config.json, sino en un archivo aparte: rutas/theme.json. Son 12 nombres de color (fondo, texto, botones, bordes...) repetidos una vez para light y otra para dark — la web cambia de uno a otro según el móvil u ordenador de quien la visita.

  1. 1Abre rutas/theme.json con el Patrón A.
  2. 2Cambia el valor hexadecimal del color que quieras, dentro de "light" o "dark" según toque.Formato siempre "#rrggbb", con la almohadilla y las comillas.
−"primary": "#2563eb",
+"primary": "#c2410c",
Un color mal elegido puede dejar texto ilegible (por ejemplo, letra clara sobre fondo claro). Antes de cambiar varios a la vez, revisa qué controla cada uno de los 12: referencia de colores ↗, con una miniatura señalando dónde se ve cada uno.

Si algo falla

Los casos más habituales, y qué hacer en cada uno.

La web deja de cargar rutas Casi siempre es un config.json mal formado: una coma de más, una de menos, o una comilla sin cerrar. Entra en rutas/config.json → History → abre la versión anterior a tu último cambio → ... ▾ → Restore this file (Patrón D). Vuelve a intentar el cambio con más cuidado.
No veo mi ruta nueva Comprueba en Actions que el despliegue terminó con un check verde ✅ (puede tardar 1-2 min). Luego recarga forzado con Ctrl Shift R. Si sigue sin salir, revisa que el archivo esté dentro de rutas/<TIPO> y no suelto en otra carpeta.
No me aparece el botón verde de commit Tu cuenta de GitHub no tiene permiso de escritura sobre el repositorio todavía. Pide que te añadan en Settings → Collaborators (ver Antes de empezar).
Subí un archivo con el nombre equivocado GitHub no deja renombrar arrastrando: sube el archivo bueno con el Patrón B y borra el antiguo con el Patrón C — dos commits, o el mismo commit si subes y borras en la misma pantalla de edición.
Hice un cambio y quiero deshacerlo Cualquier archivo de texto se puede devolver a una versión anterior sin arreglar nada a mano — ver el bloque "Deshacer" del Patrón D.
